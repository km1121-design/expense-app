# 経費申請アプリ

ブラウザで動作する経費申請アプリです。`expense-app/index.html` を開くだけで使えます
（ビルド不要）。申請データを **Google スプレッドシート（DB）** に、領収書画像を
**Google ドライブ** に保存し、実績管理・分析ツールと連携できます。

仕様の詳細は `docs/expense-app-spec.md`、バックエンドの構築手順は
`apps-script/README.md` を参照してください。

## 主な機能

### 1. レシート画像の自動読み取り（画像解析）
- レシート・領収書の写真をアップロード（クリック選択／ドラッグ＆ドロップ）。
- [Tesseract.js](https://tesseract.projectnaptha.com/) による **端末内OCR** で
  金額（合計）・日付・支払先（店名）を推定し、申請フォームへ自動入力。
- 読み取ったテキスト全文も確認可能。抽出結果は手修正できます。

### 2. 経費の申請
- 日付・科目・支払先・金額・摘要を入力して申請。
- 科目：交通費／交際費／会議費／消耗品費／通信費／宿泊費／その他。

### 3. ログイン・権限（クラウド連携時）
- ユーザーID＋パスワードでログイン。権限は **一般**（自分の申請のみ）と
  **管理者**（全件閲覧・承認・ユーザー管理）の2種類。
- 初回接続時に最初の管理者アカウントを作成。以降のユーザー追加・無効化・
  パスワード再設定は管理者ダッシュボードの「ユーザー管理」から。
- 申請者名・承認者名はサーバー側でログインセッションから強制（なりすまし防止）。

### 4. ダッシュボード（個人確認用・管理者確認用で分離）
- **個人ダッシュボード**：自分の申請一覧・件数・申請中／承認済み金額。取消・
  パスワード変更。
- **管理者ダッシュボード**（管理者のみ表示）：全申請の承認／却下／差戻、
  承認待ち・科目別金額の集計、申請者・店名検索、CSV書き出し、ユーザー管理。

### 5. クラウド連携（スプレッドシート＋ドライブ）
- 右上「⚙️」で Google Apps Script Web アプリの URL を設定すると、申請データが
  スプレッドシートに、領収書画像がドライブに保存されます。
- スプレッドシートが正本となり、**複数端末・分析ツールから同じデータを参照**できます。
- 承認・却下もスプレッドシートへ反映。オフライン時はローカルに退避し「再同期」で送信。
- 未設定の場合は認証なしの試用モード（この端末の localStorage にのみ保存）。

## 使い方

1. `expense-app/index.html` をブラウザで開く（公開URLでも可）。
2. 右上「⚙️」でクラウド連携を設定（手順は `apps-script/README.md`）。
3. 初回は管理者アカウントを作成、以降はログイン。
4. 「経費を申請」タブでレシート画像をアップロード → 自動入力を確認して申請。
5. 「個人ダッシュボード」で自分の申請状況を確認。
6. 管理者は「管理者ダッシュボード」で承認／却下・ユーザー管理。

## 実装メモ
- フレームワーク不要のバニラ JS（`index.html` / `style.css` / `script.js`）。
- クラウド連携時はスプレッドシートが正本、localStorage はキャッシュ／再送信キュー。
- OCR ライブラリと（初回の）言語データは CDN から取得するため、解析時はネット接続が必要。
- バックエンド（`apps-script/Code.gs`）は Google 環境で動作し、サーバー構築は不要。
