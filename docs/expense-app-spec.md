# 経費申請アプリ 仕様書

最終更新: 2026-07-17

## 1. 目的

経費の申請から確認・承認までをブラウザ上で完結させ、申請データを
**Google スプレッドシートにデータベース化**、**領収書画像を Google ドライブに保存**する。
ログインと権限管理（一般／管理者）を備え、蓄積データを実績管理・分析ツールから
読み込んで連携・分析できる状態にする。

## 2. システム構成

```
[利用者のブラウザ]                        [Google クラウド]
  経費申請アプリ (静的サイト)               ┌───────────────────────────┐
   - ログイン / セッション管理             │ Apps Script Web App       │
   - 画像OCR (Tesseract.js / 端末内)  ─┐   │  認証・権限チェック        │
   - 申請フォーム                      │   │  doPost: auth/create/     │
   - 個人 / 管理者ダッシュボード        ├──▶│   update/delete/users     │
   - localStorage キャッシュ           │   │  doGet : JSON（権限別）    │
                                       │   └───────┬───────────┬───────┘
                                       │           ▼           ▼
                                       │   スプレッドシート   Google ドライブ
                                       │   expenses / users  (領収書画像)
                                       │           │
[分析・実績管理ツール] ◀────────────────┘           │
   Looker Studio / BIツール / スプレッドシート集計 / JSON API / CSV
```

- フロントエンド: バニラ JS（`expense-app/`）。GitHub Pages で公開。レスポンシブ対応。
- バックエンド: Google Apps Script Web App（`apps-script/Code.gs`）。サーバー・GCP設定不要。
- データストア（1つのスプレッドシート内）:
  - `expenses` シート = 経費データ（1申請1行）
  - `users` シート = ユーザーアカウント（パスワードはソルト付き SHA-256 ハッシュ）
  - Google ドライブ「経費領収書」フォルダ = 領収書画像（行に URL を保持）

### 2.1 動作モード

| モード | 条件 | 認証 | 保存先 |
| --- | --- | --- | --- |
| クラウド | ⚙️でWebアプリURLを設定 | ログイン必須 | スプレッドシート＋ドライブ（正本） |
| ローカル（試用） | 未設定 | なし（氏名入力のみ） | localStorage のみ |

クラウドモードで通信失敗した申請は端末内の再送信キューに退避し、「再同期」で反映する。

## 3. 認証・権限

### 3.1 認証方式
- ユーザーID＋パスワード。パスワードは `users` シートにソルト付き SHA-256 で保存。
- ログイン成功で **HMAC-SHA256 署名付きセッショントークン**（有効期限12時間）を発行。
  以降のリクエストはトークンを必須とし、サーバー側で毎回検証する。
- 初回接続時（`users` が空）は「初期設定」画面で最初の管理者を作成する
  （それまでは従来互換のオープンモード）。

### 3.2 権限（ロール）

| 操作 | 一般（user） | 管理者（admin） |
| --- | --- | --- |
| 経費の申請 | ✅（申請者名はセッションからサーバーが強制） | ✅ |
| 申請の閲覧 | 自分の申請のみ | 全申請 |
| 承認 / 却下 / 差戻 | ❌ | ✅（承認者名もサーバーが強制） |
| 申請の取消 | 自分の「申請中」のみ | 全件 |
| ユーザー管理（追加・無効化・PW再設定） | ❌ | ✅（自分自身の無効化・降格は不可） |
| パスワード変更（自分） | ✅ | ✅ |

## 4. 機能要件

### 4.1 画像解析（レシートOCR）
- レシート／領収書画像をアップロード（クリック選択・ドラッグ＆ドロップ）。
- Tesseract.js による**端末内OCR**。抽出: 金額（「合計」優先）・日付・支払先。
- 抽出結果はフォームへ自動入力され、手修正可能。全文テキストも確認可能。

### 4.2 経費申請
- 入力: 日付（必須）、科目（必須）、支払先、金額（必須・1円以上）、摘要。
- 科目: 交通費／交際費／会議費／消耗品費／通信費／宿泊費／その他。

### 4.3 個人ダッシュボード
- 自分の申請一覧・集計（申請件数、申請中、承認済み金額、申請中金額）。
- 申請中の取消、領収書リンク、却下理由の確認、パスワード変更。

### 4.4 管理者ダッシュボード
- 全申請の一覧・検索（申請者・店名）・状態フィルタ・CSV書き出し。
- 集計: 総件数、承認待ち件数・金額、承認済み金額、科目別承認済み金額グラフ。
- 承認 / 却下（理由入力）/ 差戻。
- ユーザー管理: 一覧・追加（ID/表示名/初期PW/権限）・無効化/有効化・PW再設定。

### 4.5 UI
- レスポンシブ対応（モバイル〜デスクトップ）。ライトテーマ。
- デザインは今後さらに改善予定（機能を優先して構築）。

## 5. データモデル

`expenses`: `id, createdAt, applicant, date, category, vendor, amount, description,
status, reviewedAt, reviewer, reviewComment, imageUrl, imageFileId, applicantId`

`users`: `username, displayName, passwordHash, salt, role, active, createdAt`

- `status`: `pending` / `approved` / `rejected`。`role`: `user` / `admin`。

## 6. API（POST は text/plain の JSON。認証系以外は token 必須）

| action | 権限 | 内容 |
| --- | --- | --- |
| `status` | 不要 | 認証が有効か（初期設定済みか）を返す |
| `setup` | 不要（初回のみ） | 最初の管理者を作成しトークン発行 |
| `login` | 不要 | 認証してトークン発行 |
| `me` | user | セッション検証・ユーザー情報 |
| `changePassword` | user | 自分のパスワード変更 |
| `create` | user | 申請作成（画像base64があればDrive保存） |
| `update` | admin | 承認・却下・差戻 |
| `delete` | user/admin | 取消（userは自分の申請中のみ） |
| `listUsers` / `upsertUser` | admin | ユーザー管理 |
| GET `?token=` | user/admin | JSON取得（userは自分の分のみ） |

分析ツール向け: スクリプトプロパティ `SHARED_TOKEN` を設定すると、
GET `?token=<SHARED_TOKEN>` で全件を読み取り専用取得できる（Looker Studio等の定期取得用）。

## 7. セキュリティ / 制約

- パスワードは平文保存しない（ソルト付きSHA-256）。トークンはHMAC署名・12時間で失効。
- 申請者名・承認者名はサーバー側でセッションから強制し、なりすましを防止。
- スプレッドシート・ドライブ自体の閲覧権限は Google 側の共有設定に従う
  （管理者=シート所有者のみが直接閲覧可能な構成を推奨）。
- ローカルモードは試用向けで認証なし。データは端末・ブラウザ間で共有されない。
- OCR は端末内処理。申請時に画像・内容は自組織の Google 環境へ送信される。
